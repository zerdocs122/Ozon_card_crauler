# Ozon Search Position Parser

Скрипт определяет позицию товара в поисковой выдаче Ozon по поисковому запросу и артикулу (SKU). Просматривает до 100 первых результатов (3 страницы).

## Зависимости

```
pip install -r requirements.txt
```

Для работы необходим установленный **Google Chrome**.

## Использование

```bash
python parser.py "поисковый запрос" АРТИКУЛ [--headless] [--debug] [--output result.json]
```

| Флаг | Описание |
|---|---|
| `--headless` | Запустить браузер в фоне без окна |
| `--debug` | Сохранять скриншот каждой страницы |
| `--output FILE` | Сохранить результат в JSON-файл |

## Пример

```bash
python parser.py "термос" 540109988 --output result.json
```

**Пример вывода:**

```json
{
  "query": "термос",
  "sku": "540109988",
  "position": 17,
  "page": 1,
  "total_checked": 100,
  "timestamp": "2026-06-06T14:30:00"
}
```

Если товар не найден в первых 100 результатах, `position` равен `"not_found"`.

## Тестовые данные

| Запрос | Артикул (SKU) | Почему выбран |
|---|---|---|
| `термос` | `540109988` | Высококонкурентная категория, товар присутствует в выдаче — удобно проверять нахождение на конкретной позиции |
| `нож туристический` | `726417822` | Средняя конкурентность, специфичный запрос — проверяет точность извлечения SKU из URL |
| `футболка` | `2141577681` | Максимально широкая категория с тысячами товаров — проверяет сценарий `not_found` в первых 100 результатах |

## Как это работает

- Использует **nodriver** — управляет реальным Chrome без детектируемых следов автоматизации
- Обходит Cloudflare и антибот-защиту Ozon благодаря запуску настоящего браузера
- Прокручивает страницу для подгрузки lazy-loaded карточек перед сбором SKU
- SKU извлекается из URL карточки (`/product/название-товара-АРТИКУЛ/`) прямо в JavaScript
